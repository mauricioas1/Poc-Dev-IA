# Contribuindo

Obrigado por querer contribuir com este projeto. Siga estas diretrizes:

- Fork o repositório e crie uma branch descritiva: `feature/<descrição>` ou `fix/<descrição>`.
- Sempre abra Pull Requests contra `main` com uma descrição clara e um changelog resumido.
- Execute os testes locais antes de abrir PRs:

```bash
mvn test
```

- Formato de commits: usar mensagens no estilo Conventional Commits (ex.: `feat:`, `fix:`, `chore:`).
- Código Java deve seguir as convenções do projeto e passar o linter/formatter da equipe.
